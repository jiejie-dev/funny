# Funny v2.1.2 — VM Control Flow and Stdlib Parity

**Date:** 2026-07-07

v2.1.2 closes the VM-path gaps called out in the v2.1.0 release blog: `match`, `break`/`continue`, and `x in list` now work on the default bytecode path without `FUNNY_INTERPRET=1`.

## Highlights

- **`match` / `break` / `continue`** — full end-to-end on the VM (type checker, compiler, VM)
- **`x in list`** — new `IN_LIST` opcode on the default path
- **Stdlib parity** — interpreter and VM share `internal/stdlib.Call` (33 builtins on both paths)
- **Top-level `println(...)` type-checked** — undefined calls fail at compile time

## Install

```bash
go install github.com/jiejie-dev/funny/v2/cmd/funny@v2.1.2
```
