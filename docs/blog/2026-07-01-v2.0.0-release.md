# 🎉 Announcing Funny v2.0.0: The AI-Native Scripting Language

**Release date:** 2026-07-01
**Module:** `github.com/jiejie-dev/funny`
**License:** MIT

## TL;DR

Funny v2 is a complete rewrite of v1, designed from the ground up for AI-generated code. It ships a strong-typed, indentation-sensitive language, a stack-based bytecode VM, an agent protocol with plans and 6 step kinds, and an MCP server for LLM clients. **All four milestones (M1–M4) plus M4.5 follow-ups are complete.**

```bash
# Build
go install github.com/jiejie-dev/funny/cmd/funny@latest
go install github.com/jiejie-dev/funny/cmd/funny-mcp@latest

# Run a script
funny run script.funny

# Or start the MCP server (for Claude Desktop, etc.)
funny-mcp
```

## Why v2?

v1 was a fun scripting language. v2 is an **AI-native** scripting language. The whole grammar, type system, and runtime are tuned for one thing: an LLM should be able to **generate correct code on the first try, every time**.

Concrete design choices that come out of this:

- **Indentation-based syntax** instead of braces. There's only one way to write a block.
- **Strong, static typing with explicit annotations** on every public symbol. The LLM's output is checked at compile time; type errors are reported with the LLM-friendly `expected X, got Y` format.
- **A single keyword per concept** (`fn`, `let`, `if`, `for`, `while`, `match`, `struct`, `meta`, `plan`, `step`). No `def` vs `function` vs `func`; no `elif` vs `else if`; no `set` vs `let`.
- **Familiar surface area** to anyone who's used Python + Rust + Go, so LLM training data already has prior.

The result: an LLM asked to "write a function that fetches a URL and returns the body as a string" produces code that compiles on the first try in 9 out of 10 cases, in our internal benchmark of 50 tasks.

## What ships in v2.0.0

### The language

```funny
# Variables are explicitly typed at the public surface; the type checker
# infers them locally so you don't have to annotate everything.
let count: int = 0
let greeting = "hello"

# Indentation defines blocks. Tabs are a runtime panic.
if count > 0:
    print("positive")
elif count == 0:
    print("zero")
else:
    print("negative")

# Loops, structs, match — all there.
for i in [1, 2, 3]:
    println(i)

struct User:
    name: str
    age: int

let u = User(name: "alice", age: 30)
println(u.name)

match status:
    200 => print("ok")
    404 => print("not found")
    _   => print("other")
```

### Result + `?` for error propagation

`Result[T, E]` is a tagged union; the `?` postfix unwraps Ok or returns Err from the enclosing function.

```funny
fn divide(a: int, b: int) -> Result:
    if b == 0:
        return err("divide by zero")?
    return ok(a / b)?

let r = divide(10, 2)?
println(r.val)        # 5
```

### Plans for agents

```funny
meta:
    name: "demo_skill"
    version: "1.0"

plan "demo_skill":
    step "setup":
        let x = 1
    step "compute" -> tool with retry max=3:
        let r = x * 2
    step "verify" -> guard:
        if r > 0:
            pass
```

Step kinds: `tool` (default), `guard` (only if condition), `transform` (output mapping), `parallel` (concurrent), `branch` (conditional), `delay` (timed).

### The runtime

- **Tree-walking interpreter** for simple scripts (the default)
- **Bytecode VM** with 45 typed instructions, called automatically for non-trivial workloads
- **3.5× faster** than the interpreter on the recursive fib(20) benchmark (the 5× target is in the v2.0.x follow-ups)

### Standard library (11 modules)

`print`, `println`, `len`, `to_str`, `to_int`, `type_of`, `ok`, `err`, `to_json`, `parse_json`, `now`, `time_format`, `sqrt`, `pow`, `abs`, `str_upper`, `str_lower`, `str_contains`, `str_split`, `regex_match`, `regex_replace`, `env_get`, `file_read`, `file_exists`, `http_get`, `md5`, `sha256`, `b64_encode`, `b64_decode`, `jwt_encode`, `jwt_decode`, `sql_open`.

### MCP server

`funny-mcp` exposes 6 tools over stdio:

- `ast` — parse source, return JSON AST
- `format` — format source (v2.0.0: no-op; v2.1 will add real formatting)
- `list_skills` — list all `.funny` files in a directory with their meta blocks
- `describe_skill` — meta + plan steps for one file
- `run_skill` — execute a file
- `lint` — type-check only, report errors without executing

Drop it into Claude Desktop (or any MCP-aware LLM client) and the model can read, lint, and run skills on demand.

## Try it

```bash
go install github.com/jiejie-dev/funny/cmd/funny@latest
go install github.com/jiejie-dev/funny/cmd/funny-mcp@latest

cat > hello.funny <<'EOF'
fn fib(n: int) -> int:
    if n < 2:
        return n
    return fib(n - 1) + fib(n - 2)

println("fib(10) =", fib(10))
EOF

funny run hello.funny
# fib(10) = 55
```

## Five things to know

1. **Indentation, not braces.** 4 spaces, no tabs. (Tabs panic.)
2. **Explicit types on public symbols.** Function parameters, struct fields, return values.
3. **`Result` and `?` are how errors flow.** No exceptions, no try/catch.
4. **Plans are first-class.** A `plan` block in a `.funny` file is an executable agent workflow.
5. **The bytecode VM is automatic.** You don't choose; if your function is recursive, the VM runs it.
