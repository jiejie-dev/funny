# Funny v2.4.1 — Plan Timeout Preemption

**Date:** 2026-07-07

v2.4.1 fixes a long-standing plan engine caveat: step `with timeout` now truly stops runaway evaluator work.

## Highlights

- **Cancellable evaluator** — context checked at loop heads and statement boundaries
- **No background scope mutation** — timed-out `while true` loops stop instead of racing later steps
- **Same timeout syntax** — `with timeout="500ms"` on any step kind

## Install

```bash
go install github.com/jiejie-dev/funny/v2/cmd/funny@v2.4.1
```

## Before vs after

**Before (v2.4.0):** timeout returned an error but the goroutine kept running and could keep writing plan variables.

**After (v2.4.1):** deadline fires → context cancelled → evaluator exits at next preemption point → plan continues safely.
