# Funny v2.4.2 — REPL/LSP Experience Unification

**Date:** 2026-07-07

v2.4.2 aligns interactive tooling with the default execution path: the REPL runs on the bytecode VM, and LSP hover surfaces `##` documentation.

## Highlights

- **REPL → VM** — persistent sessions compile and run on the same backend as `funny run`
- **Doc hover** — `fn` and `struct` hovers include summary, argument notes, and return descriptions from `##` comments
- **Interpreter fallback** — `FUNNY_REPL_INTERPRET=1` preserves the previous tree-walking REPL

## Install

```bash
go install github.com/jiejie-dev/funny/v2/cmd/funny@v2.4.2
```

## Note

The REPL re-runs all prior cells on each input (side effects such as `println` repeat). Use the interpreter fallback if you need the old per-cell semantics.
